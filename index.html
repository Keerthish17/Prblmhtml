Category 1 · Levels 1–3

Subarray Choice → Maximize

The most fundamental optimization pattern. You choose a contiguous subarray. The question is: which contiguous window gives the best result? The insight is always the same: when does extending the window help vs hurt?

Kadane's Algorithm Running Contribution Reset When Negative
REUSABLE TEMPLATE — Subarray Optimization
# The universal Kadane template for ANY subarray optimization
best = -infinity          # global answer
current = 0               # contribution of current window

for each element x in array:
    current = COMBINE(current, x)   # extend window: what does x add?
    best = max(best, current)       # is this the best we've seen?
    if SHOULD_RESET(current):       # does carrying this window HURT?
        current = IDENTITY          # start fresh

# For MAX SUM:    COMBINE = +,   RESET when current < 0, IDENTITY = 0
# For MAX PRODUCT: COMBINE = ×,  track BOTH max and min (negatives flip!)
# For CIRCULAR:   run twice — once normal, once on negated array

Maximum Subarray — LC 53

Objective: choose a contiguous subarray · maximize its sum · Hard edge: all negative
Easy-MediumKadaneContribution
Problem

Given array nums, find the contiguous subarray with the largest sum. The subarray must contain at least one element.

🧠 Run The Mental Model First

1

Objective? Maximize sum of a chosen contiguous window.

2

Allowed move? Extend the current window right by one element, OR start a fresh window at current element.

3

Contribution of extending? +nums[i] to the current window sum.

4

Gain or loss? If current sum before adding is negative, it HURTS the new element. Carrying a negative prefix is always worse than starting fresh.

5

Best local choice? current = max(nums[i], current + nums[i]). Equivalently: reset if current drops below 0.

6

Global optimum? Track the max across all local bests.

⚡ AHA Moment

"A negative prefix is a liability, not an asset. If carrying the window so far makes the current element smaller than it is alone, abandon the old window. The moment current sum goes negative, you're better off starting fresh — because any future element added to a negative base is worse than that element alone."

Let's use: nums = [-3, 1, -2, 4, -1, 2, 1, -5, 4, -8, 3]

Maximum Product Subarray — LC 152

Allowed move: extend window · Contribution: multiply · Twist: negatives flip sign!
MediumTrack Min+MaxNegatives Flip

🧠 Mental Model

1

Objective? Maximize product of a contiguous subarray.

2

Contribution? Multiplying by nums[i]. But here's the twist — a NEGATIVE number FLIPS the sign.

3

Critical insight: The current MINIMUM (most negative) × a negative number = a very large POSITIVE. So we must track BOTH max and min product ending at each position.

4

Best choice? At each step: new_max = max(nums[i], cur_max×nums[i], cur_min×nums[i]). Same for new_min.

⚡ AHA Moment

"In product problems, the most NEGATIVE running product is secretly your best friend — one more negative number and it becomes the largest positive. You must carry BOTH the maximum and minimum product at every step. This is the only difference from sum Kadane."

Maximum Sum Circular Subarray — LC 918

Same as L1 but the subarray can wrap around the array ends
Medium-HardKadane TwiceComplement Trick

🧠 Mental Model

1

Two cases exist: The best subarray either (A) doesn't wrap around → normal Kadane, OR (B) wraps around → it skips a middle chunk.

2

Case B insight: If the best subarray wraps, then the COMPLEMENT (the middle chunk that's NOT included) must be the MINIMUM subarray. So: circular best = total_sum - min_subarray_sum.

3

Answer = max(case A, case B). But if all elements are negative, case B = 0 which is wrong → answer is just case A.

⚡ AHA Moment

"A circular subarray that wraps = total array MINUS the middle part you skip. Maximizing what you include = minimizing what you exclude. So run Kadane TWICE: once for max subarray, once for min subarray. The circular answer is total - min_subarray."

Complex Test Case

Category 2 · Levels 4–6

Two Pointer → Maximize Area / Volume

You have two boundaries. Moving them changes the contribution. The pattern: move the pointer that currently limits you — never the one that helps you.

Two Pointer Contribution = f(height, width) Move the bottleneck
REUSABLE TEMPLATE — Two Pointer Optimization
left, right = 0, n-1
best = 0

while left < right:
    contribution = COMPUTE(left, right)   # what does this pair contribute?
    best = max(best, contribution)
    
    # KEY INSIGHT: move the side that LIMITS you
    if LIMITS(left, right):               # which side is the bottleneck?
        left += 1                         # moving left might find better
    else:
        right -= 1                        # moving right might find better

# For Water: contribution = min(h[l],h[r]) × (r-l). Bottleneck = shorter wall.
# For Rain:  contribution is per-bar. Bottleneck = min of left_max, right_max.

Container With Most Water — LC 11

Move two walls. Water = min(height) × width. Always move the shorter wall.
MediumTwo PointerBottleneck Logic

🧠 Mental Model

1

Contribution of a pair (l,r)? Water = min(h[l], h[r]) × (r - l).

2

What limits us? The SHORTER of the two walls. The taller wall is irrelevant beyond a certain height.

3

If we move the taller wall inward? Width decreases AND height can only stay same or decrease (limited by same short wall). Strictly worse or equal. Never better.

4

If we move the shorter wall inward? Width decreases BUT we might find a taller wall → could be better. This is our only hope.

5

Greedy:? Always move the shorter wall. We never miss the optimal pair.

⚡ AHA Moment

"Moving the TALLER wall can NEVER improve the answer — width shrinks and the height is still capped by the SHORT wall. So moving the taller wall is always useless. We ONLY move the shorter wall because it's our only chance to escape the current bottleneck. This greedy is provably correct."

Each bar can be the height of a rectangle. How wide can it extend? Find its contribution.
HardMonotonic StackContribution Per Bar

🧠 Mental Model

1

What contributes? Each bar can be the HEIGHT of the largest rectangle it belongs to.

2

How wide can bar i extend? Left until we hit a bar shorter than h[i]. Right until we hit a bar shorter than h[i].

3

Contribution of bar i: h[i] × (right_boundary - left_boundary - 1).

4

Insight: A bar "dies" (its width is bounded) the moment a SHORTER bar appears. A monotonic increasing stack processes each bar exactly once and pops it when its right boundary is found.

⚡ AHA Moment

"Every bar wants to extend as far as possible. It can only extend until it meets a shorter bar. A monotone stack gives us the EXACT moment each bar's right boundary is hit — when we pop a bar, we've just found its right wall. The left wall was the new stack top. This processes n bars in O(n)."

Maximum Swap — LC 670

Exactly 1 swap of two digits. Find the pair that gives maximum number.
MediumGreedy DigitLast Occurrence

🧠 Mental Model

1

Contribution of a swap? Swapping digit at position i with digit at position j changes the value at position i (more significant) by (d[j]-d[i]) × 10^(n-1-i).

2

Best gain? We want the LEFTMOST position i where we can place a LARGER digit from the RIGHT. From the right, we want the LARGEST digit, and if there are ties, the RIGHTMOST one (so we swap with the least significant occurrence of the maximum).

3

Why rightmost occurrence of max? If digit 9 appears at positions 3 and 7, swapping with position 7 leaves the 9 at position 3 untouched and still gives us a 9 in front.

⚡ AHA Moment

"The most valuable swap places the largest possible digit as far LEFT as possible (most significant position). Scan left to right — at each position, check if any digit to its RIGHT is larger. Among all larger digits to the right, pick the largest one, and if tie, the rightmost. This greedy is O(n) using a last-occurrence map."

Complex Test Case — Multiple Traps

Case 1: num = 98368

Digits: [9,8,3,6,8]. The 9 is already first. Try to help 8: can we swap 8 at pos 1 with a later 8 at pos 4? No gain (8↔8). Try to help 3 at pos 2: largest digit to right is 8 (at pos 4). Swap pos 2 and 4: 98863 → gain!

1884No (tie, but rightmost 8 = pos 4 = same value)

Result: 98863 = 98863

Case 2 (Tricky): num = 1993

Digits: [1,9,9,3]. Last occurrence: {1:0, 9:2, 3:3}. Scan: pos 0 has digit 1. Max to right = 9 at pos 2 (rightmost 9). Swap pos 0 and pos 2: [9,9,1,3] = 9913.

Note: we pick pos 2 not pos 1 even though both are 9 — rightmost occurrence means if we swap with pos 1, the 9 at pos 2 is still there. Result is the same. Both give 9913. ✓

Case 3 (Already Max): num = 9973 → No larger digit to right of any position → no swap helps → return 9973.

⚠ Edge Cases
Single digit: 5 → no swap possible, return 5.

Maximize Sum After K Negations — LC 1005

K negations. Negate the most negative first. Handle odd K leftover.
EasySort + GreedyParity of K

🧠 Mental Model

1

Contribution of negating element x? Sum changes by -2x. If x is negative, -2x is positive → gain! If x is positive, -2x is negative → loss!

2

Greedy: Always negate the most negative element first — maximum gain per operation.

3

What if K left after all negatives gone? We must negate positives. To minimize loss: negate the SMALLEST positive. And if K leftover is EVEN → two negations cancel → no net effect on sum.

⚡ AHA Moment

"Sort by value. Negate from most negative. Once you run out of negatives, an even number of remaining operations = free (cancel out). Odd remaining = must negate the element closest to zero (minimum |value|) once. The key question is: after greedily handling negatives, what's the PARITY of remaining K?"

Minimum Swaps to Make Strings Equal — LC 1247

Two strings of x's and y's. Swap chars across strings to make them equal.
MediumMismatch CountingParity Insight

🧠 Mental Model

1

What are the mismatches? At each position i, if s1[i]≠s2[i], it's a mismatch. Two types: (s1='x',s2='y') call it XY, and (s1='y',s2='x') call it YX.

2

One swap fixes two XY mismatches: take an 'x' from s1[i] and swap with 'y' in s1[j] where both positions are XY-type. Costs 1 swap.

3

One swap fixes two YX mismatches: similarly costs 1 swap.

4

One XY + one YX leftover? Needs 2 swaps to fix.

5

If (XY + YX) is odd → impossible (total chars of x and y can't balance).

⚡ AHA Moment

"Don't simulate swaps. Count mismatch TYPES. Two same-type mismatches cancel with ONE swap. Two different-type mismatches need TWO swaps. The answer = (xy//2) + (yx//2) + 2*(xy%2). The remainder is always 0 or 1 for each, and if both have remainder 1, that costs 2 extra swaps."

Remove / Delete → Minimize / Validate

You can remove K elements. Each removal contributes to the goal. The pattern: always remove the element that hurts you most right now — greedy with a monotonic stack.

Monotonic Stack Greedy Remove What stays = what's removed
REUSABLE TEMPLATE — Greedy Removal
stack = []
k_remaining = k

for each element x:
    # While we still have removals and the top of stack is "worse":
    while k_remaining > 0 and stack and SHOULD_REMOVE(stack[-1], x):
        stack.pop()          # remove the bad element
        k_remaining -= 1
    stack.append(x)

# If k_remaining > 0: remove from the END (or handle remainder)
result = stack[:len(stack) - k_remaining]

# For Remove K Digits: SHOULD_REMOVE = stack[-1] > x (want smaller digits first)
# For Duplicate Letters: SHOULD_REMOVE = stack[-1] > x AND stack[-1] appears later
# For Parentheses: SHOULD_REMOVE = unmatched bracket

Remove exactly K digits to form the smallest possible number.
MediumMonotonic StackGreedy

🧠 Mental Model

1

2

3

⚡ AHA Moment

"We want the SMALLEST number. Smaller digits should be at the FRONT. The monotone stack keeps digits in non-decreasing order, popping (removing) any digit that's larger than what just arrived."

Remove Duplicate Letters — LC 316

Remove duplicates so each letter appears once. Result must be lexicographically smallest.
Medium-HardStack + Last OccurrenceGreedy

🧠 Mental Model

1

Same stack logic as L10 BUT we can only remove a letter from stack if it appears AGAIN later (otherwise it's our only chance to include it).

2

Contribution of keeping vs removing letter c at top? If c > current and c appears again later → pop and remove c. If c won't appear again → MUST keep it.

3

Visited set: Don't add a letter that's already in the stack (we have its best position already).

Master Cheat Sheet — Study This Before Any Interview

Every pattern, every trigger, every template. One page to rule them all.

THE 6-QUESTION CHECKLIST — Run This Before Touching Code
Questions to Ask
  • What is the OBJECTIVE? (maximize, minimize, return)
  • What is ONE allowed MOVE/OPERATION?
  • What does ONE move CHANGE? What stays fixed?
  • What is the CONTRIBUTION of this move?
  • What makes one move BETTER than another?
  • Is order of moves important? (Greedy) or Do we need all options? (DP)
Red Flags = Use These Patterns
  • "Contiguous subarray" → Kadane / sliding window
  • "Two boundaries / walls" → Two pointer
  • "K digits/elements to remove" → Mono stack
  • "K ops each affecting a range" → Diff array + greedy
  • "Minimize maximum / maximize minimum" → Binary search on answer
  • "Merge/split cost" → Min-heap (Huffman)

📌 Pattern 1 — Kadane (Subarray Max/Min)

Trigger: "contiguous subarray", "maximize sum/product"

Template:
cur=0; best=-inf
for x: cur=max(x, cur+x)
best=max(best,cur)

Twist for product: Track both cur_max AND cur_min — negatives flip!

Twist for circular: Answer = max(normal_kadane, total - min_kadane)

📌 Pattern 2 — Two Pointer (Area/Volume)

Trigger: "two walls", "maximize area", "trapped water"

Template:
l,r=0,n-1
while l<r:
compute(l,r)
if h[l]<h[r]: l++
else: r--

Key: Always move the LIMITING (shorter) side. Moving the stronger side never helps.

📌 Pattern 3 — Monotonic Stack (Remove/Contribute)

Trigger: "remove K digits", "lex smallest", "rectangle in histogram"

Template:
stk=[]; k_rem=k
for x in arr:
while k_rem and stk and stk[-1]>x:
stk.pop(); k_rem-=1
stk.append(x)

For duplicates: Add "AND has_later_occurrence(top)" condition before popping.

📌 Pattern 4 — K Transformation Greedy

Trigger: "K negations/swaps", "maximize/minimize after K ops"

Template:
sort by gain (desc)
apply k ops greedily
if k leftover:
even→no change
odd→negate min|x|

Key: Contribution = change in objective per operation. Sort by contribution descending.

📌 Pattern 5 — Diff Array (Range Ops)

Trigger: "flip window of K", "range updates", "consecutive bit flips"

Template:
diff=[0]*(n+1); flips=0
for i: flips+=diff[i]
if (val+flips)%2!=want:
diff[i+k]-=1; flips+=1
ops+=1

Key: Never simulate the range. Record start/stop in diff array. Prefix sum reconstructs.

📌 Pattern 6 — Binary Search on Answer

Trigger: "minimize maximum", "maximize minimum", K groups

Template:
lo=min, hi=sum
while lo≤hi:
mid=(lo+hi)//2
if can(mid,k): ans=mid; hi=mid-1
else: lo=mid+1

Key: The answer is monotone. If X works, X+1 also works (for minimize-max). Binary search + O(n) greedy check.

📌 Pattern 7 — Heap Greedy (Merge/Connect)

Trigger: "minimum cost to merge/connect", "connect sticks/ropes"

Template:
heapq.heapify(arr)
cost=0
while len(heap)>1:
a=heappop(); b=heappop()
cost+=a+b
heappush(a+b)
return cost

Key: Smallest elements should be merged first — they'll participate in more merges but at lower cost.

📌 Pattern 8 — 2-State DP (Swap Sequences)

Trigger: "two sequences", "swap positions", "both increasing"

Template:
keep=0; swap=1
for i in 1..n:
new_keep=inf; new_swap=inf
if natural_ok: new_keep=min(new_keep,keep)
if swap_prev_ok: new_keep=min(new_keep,swap)
if swap_ok: new_swap=min(new_swap,swap+1)
keep,swap=new_keep,new_swap

Key: Binary state per position. Transitions only between compatible state pairs.

⚡ The Universal Insight — Burn This Into Memory

Every optimization problem has ONE key question: "What is the contribution of a single move?"

• If one move affects a contiguous range → diff array or Kadane.
• If one move eliminates the worst thing → greedy pop (stack).
• If many moves and we want the best K → sort by contribution, take top K.
• If we can't enumerate moves but can CHECK a given result → binary search + greedy validator.
• If cost depends on merge ORDER → heap (smallest contribution first).
• If two interacting choices per position → DP with 2 states.
